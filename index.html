El equipo 10 agentes especializados con modelos optimizados por tarea


dev-senior Opus
Implementa features y repara bugs con calidad de producción. Para bugs difíciles: 6 fases de diagnóstico (feedback loop primero, hipótesis, instrumentación). TDD: vertical slices, tests de comportamiento, anti-tautológicos.
hawkeye Sonnet
Dev junior del equipo. Tareas acotadas con guía explícita, confirma antes de cada acción, activa junior-code-review automáticamente.
qa-bug-hunter Sonnet
Caza bugs con herramientas de solo lectura (sin Bash) — no ejecuta, no repara, no decide. Alcance acotado al diff de la misión, reproducción descrita en texto, y escala a Nick Fury en vez de actuar.
ui-ux-designer Sonnet
Designer senior de producción. 10 categorías de prioridad (WCAG 2.2, iOS HIG, Material Design), Component Checkpoint, pre-delivery checklist. Stack fijo: Lucide, SweetAlert2, Tom Select, Flatpickr, Tippy.js, Simple-DataTables — nunca emojis ni diálogos nativos.
security-analyst Opus
Analista senior. 17 categorías de auditoría (SECRETS, DATABASE, AUTH, ACCESS_CONTROL, SSRF, XSS, PAYMENTS, DEPENDENCIES…). Workflow investigate→report→plan→implement→verify. Cubre AI/LLM keys, Supabase RLS, Firebase rules y vibe-coding patterns.
dba Sonnet
Migraciones SQL, cambios de esquema, índices y queries. Entrega archivos listos para aplicar.
dashboard-analyst Sonnet
Define QUÉ mostrar antes de diseñar: selección de KPIs, tipo de gráfico por caso de uso, jerarquía ejecutivo→operativo→analítico, principios Tufte/Few. Handoff al ui-ux-designer con especificación lista.
contador Opus
Genera estructura y código de reportes financieros: conciliación bancaria, P&L, balance general, flujo de caja, IT-1/IT-2/606/607 DGII. Base NIIF + particularidades RD (ITBIS 18%, ISR 27%). Siempre marca ⚠ revisión contable requerida.
devops Sonnet
Docker Compose, nginx/PHP-FPM, GitHub Actions CI/CD, scripts de servidor, hardening básico. Stack objetivo: Laravel + PHP-FPM + PostgreSQL + Redis. Deploy vía git pull con zero-downtime y health check.
documentalista Haiku
Registra cada misión en el vault de Obsidian: bitácora, decisiones ADR, bugs y pendientes. Fundamenta en evidencia real (git log/diff), hace updates quirúrgicos con presupuesto de cambio, y evita duplicar: un concepto, un hogar canónico.

Flujo de una misión Orquestado por Nick Fury (la skill vengadores)


1
Usuario activa la misión
Escribe /vengadores implementa el módulo de pagos o describe la tarea.
2
Nick Fury analiza y presenta el plan
Selecciona solo los agentes necesarios, define el orden y los handoffs. Espera aprobación antes de ejecutar.
3
Ejecución secuencial
Cada agente hace su parte y pasa el resultado al siguiente. Patrón típico:
qa-bug-hunter → dev-senior → dba → security-analyst → navegador-qa → testsprite → documentalista
4
Cierre en el vault
El documentalista actualiza la bitácora, ADRs, bugs y pendientes en Obsidian. Nick Fury reporta el resumen al usuario.

Verificación en dos capas Local antes del commit · en vivo después del deploy


El problema que resuelve

"Un loop sin verificador real no falla en voz alta — alucina progreso."
El equipo verifica dos veces: Playwright CLI maneja un navegador real contra tu app local antes del commit (gratis), y TestSprite corre tests contra la app desplegada después del deploy.

dev escribe → navegador-qa (local) → commit / deploy → testsprite verifica en vivo → pasa → documentalista 
# Capa 1 — Playwright CLI (local, gratis; siempre CLI, nunca el MCP)
npm install -g @playwright/cli
playwright-cli install-browser chromium
# Tras un cambio con superficie web (skill renombrada para no colisionar con el binario):
/navegador-qa

# Capa 2 — TestSprite (app desplegada, requiere URL pública)
npm install -g @testsprite/testsprite-cli
export TESTSPRITE_API_KEY=<tu-clave>   # testsprite.com → Settings → API Keys
testsprite setup --from-env --agent claude
# En cualquier misión con la app desplegada:
/testsprite

Mantené tu instalación al día Un prompt sincroniza agentes y skills con la última versión del repo


1 Copiá este prompt y pegalo en tu Claude Code

Para cuando ya instalaste Vengadores antes y solo querés lo último: agentes corregidos, skills nuevas, renombres. La IA clona o actualiza el repo, corre update.sh/update.ps1 y te cuenta qué cambió.

Quiero actualizar mi instalación del sistema Vengadores a la última versión del repo. Encargate vos — yo no voy a tocar la terminal.

1. Si existe ~/vengadores y es un repo git, entrá ahí. Si no existe, cloná primero:
   git clone https://github.com/hectorgvc/vengadores.git ~/vengadores

2. ANTES de ejecutar nada, traé la última versión del repo:
   git -C ~/vengadores pull --ff-only
   Esto garantiza que el propio script de actualización sea el más reciente (un script viejo con un bug no puede repararse a sí mismo). Si el pull falla por cambios locales, mostrame qué archivos difieren y preguntame antes de descartar nada.

3. Recién ahí corré el script de actualización según mi sistema operativo:
   - Linux/macOS/WSL2: chmod +x update.sh && ./update.sh
   - Windows (PowerShell): .\update.ps1
   En Windows: si da error de sintaxis o de caracteres raros, NO edites el script — verificá con git log que el pull del paso 2 realmente trajo lo último, y avisame.

   Este script actualiza los agentes y skills que ya tenía instalados (sobreescribe su contenido con la versión del repo), agrega lo que sea nuevo, y limpia cualquier skill renombrada u obsoleta. No toca mi vault más allá de 03-Skills/ — mis proyectos y notas quedan intactos.

4. Leeme el resumen que imprime el script: qué agentes se actualizaron, qué skills son nuevas, y si hubo alguna migración (por ejemplo, renombres de skills).

5. Si el resumen menciona que una skill fue renombrada o reemplazada, decime el nombre nuevo y cómo invocarla de ahora en más.

6. Si algo falla (el repo no existe, sin permisos, sin internet), explicámelo en español simple y decime exactamente qué tengo que resolver antes de reintentar.

No hace falta que repitas el onboarding (jarvis) a menos que yo te lo pida explícitamente — mi perfil ya está configurado.
2 Listo
